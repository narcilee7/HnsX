# SBOM & signing (v0.8 scaffold)

Production releases attach:

- A CycloneDX SBOM (`.cdx.json`) generated by `syft` against the built binary.
- A cosign signature (`.sig`) over the release tarball.
- A cosign certificate (`.pem`) for key transparency.

These files are produced by the release pipeline and uploaded alongside the
release artefacts. They are not generated from this CLI.

Verification (consumer-side):

```bash
cosign verify-blob \
  --certificate-identity-regexp 'https://github.com/narcilee7/HnsX' \
  --certificate-oidc-issuer 'https://token.actions.githubusercontent.com' \
  hnsx_0.8.0_darwin_arm64.tar.gz
```

v0.8 only documents the contract; the release pipeline wiring lands in the
follow-up commit that adds GitHub Actions workflows for `release/v*` tags.
