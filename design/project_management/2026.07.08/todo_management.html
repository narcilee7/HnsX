1. 后端 Worker（hnsx-worker）#

1.1 Worker parent 进程#

  • 实现 hnsx-worker run --server=... --worker-id=... CLI 入口
  • 通过 gRPC PullSession 长轮询从 Go Scheduler 拉取待执行 Session
  • 每 5s 发送 Heartbeat，上报 worker 状态与当前负载
  • 维护 StreamChannel 双向流，上传 Observation / 接收 Cancel

1.2 SessionRuntime 子进程#

  • 用 subprocess.Popen 隔离每个 Session 执行
  • 子进程内加载 DomainSpec 并校验 Harness 配置
  • 子进程退出时父进程回收资源，异常退出时上报 error observation
  • 收到 Cancel 信号时向子进程发送 SIGTERM，支持优雅退出

1.3 Noop adapter#

  • 实现 NoopAdapter，直接返回固定文本 observation
  • 跑通 trigger → session → turn → observation → end 的完整状态机
  • 把 observation 序列通过 StreamChannel 实时回传 Go Control Plane

1.4 测试与验收#

  • make worker-test 全部通过
  • 端到端覆盖：拉任务 → fork runtime → noop adapter → 回传 3 条以上 observation（见 scripts/smoke-worker.sh）

2. Go 服务调度（hnsx-server）#

2.1 Worker Registry#

  • 维护内存中的 worker 列表（worker-id、capabilities、last heartbeat、current session）
  • 心跳超时通过 EvictStale 自动清理离线 worker
  • 提供 Register / Heartbeat / List / Get / Inbound 接口
  • 新增 AssignSession / SessionWorker / UnassignSession，维护 session → worker 映射

2.2 Scheduler#

  • Session 触发后 Enqueue 到 SessionQueue
  • Worker 通过 PullSession 长轮询拉取任务
  • Worker 离线时自动重新调度（当前行为：等长轮询 30s 超时后重新入队；需 queue 感知 worker liveness）

2.3 gRPC 服务对接#

  • 补全 proto/hnsx/v1/worker.proto 的 Go service 实现：
    • PullSession（long-poll）
    • Heartbeat（unary）
    • StreamChannel（bidirectional streaming）
  • 与 Python worker 完成真实 gRPC 握手并跑通全链路

2.4 Cancel 传播#

  • Console / CLI 调用 Cancel Session 时，通过 Registry.SessionWorker 找到对应 worker
  • 通过 StreamChannel 下发 CancelSessionCommand
  • Python worker 收到后终止子进程并回传 cancelled observation

2.5 测试与验收#
